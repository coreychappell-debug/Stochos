### 2\. The Core Calculation Engine (Formulas)

This is the mathematical logic that will power the tool's live updates.

**Inputs:**

  * $FY\_{Sales}$ = Fiscal Year Total Sales Target (e.g., $4,800,000,000)
  * $D\_{Mix%}$ = The sales mix percentage for a given denomination (e.g., 25% for $5 games)
  * $D\_{Price}$ = The price of the denomination (e.g., $5)
  * $G\_{Payout%}$ = The average payout percentage for a game
  * $G\_{Retailer%}$ = The retailer commission percentage
  * $G\_{System%}$ = The gaming system/operator percentage
  * $C\_{Base}$ = Base manufacturing cost (either as a per-thousand rate, $C\_{M}$, or a percent of sales, $C\_{%}$)
  * $C\_{Features}$ = Sum of costs for all selected special features
  * $C\_{OneTime}$ = Sum of all one-time costs for a game

**Core Formulas:**

1.  **Denomination Revenue Target:**
    $$D_{Revenue} = FY_{Sales} \times D_{Mix\%}$$

2.  **Denomination Unit Volume:**
    $$D_{Units} = \frac{D_{Revenue}}{D_{Price}}$$

3.  **Total Game COGS (Cost of Goods Sold):**
    $$\text{COGS}_{Game} = (\text{Payout} + \text{Retailer} + \text{System} + \text{Manufacturing} + \text{Features})$$

      * Where:
          * $\\text{Payout} = D\_{Revenue} \\times G\_{Payout%}$
          * $\\text{Retailer} = D\_{Revenue} \\times G\_{Retailer%}$
          * $\\text{System} = D\_{Revenue} \\times G\_{System%}$
          * $\\text{Manufacturing} = (\\text{if using } C\_M: \\frac{D\_{Units}}{1000} \\times C\_M) \\text{ or } (\\text{if using } C\_{%}: D\_{Revenue} \\times C\_{%})$
          * $\\text{Features} = \\sum(\\text{Cost of each selected feature})$

4.  **Game Gross Margin:**
    $$\text{Margin}_{Game} = D_{Revenue} - \text{COGS}_{Game}$$

5.  **Net Margin (after one-time costs):**
    $$\text{Net Margin}_{Game} = \text{Margin}_{Game} - C_{OneTime}$$

**Primary User Flow:**

1.  **Load:** User opens the HTML file and loads a `.json` plan or starts fresh.
2.  **Set Targets:** User inputs the FY Total Sales target at the top.
3.  **Allocate Mix:** User adjusts the sliders or inputs percentages in the **Left Panel**.
4.  **Build Plan:** User clicks "[ADD GAME]" in the **Center Panel** for a specific denomination.
5.  **Define Game:** User names the game, assigns a vendor, and toggles required features.
6.  **Input Costs:** In the **Right Panel** (not shown in wireframe), the user selects the vendor tab and inputs the contract pricing for the selected features.
7.  **Review:** The tool instantly recalculates all totals, margins, and charts. Risk flags appear if margins are negative or data is missing.
8.  **Save/Export:** User saves the plan as a `.json` file or exports reports as `.csv` or `.xlsx`.
